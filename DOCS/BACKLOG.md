# BACKLOG DOCUMENTAL AIAGRO

## GOV-DOC-001 — Plantilla estándar de evidencia de auditoría

**Estado:** Pendiente
**Prioridad:** Media
**Origen:** CTO / Dominio 09
**Relacionado con:** GOV-PROCESS-001 / GOV-SEC-001

### Objetivo

Crear `DOCS/TEMPLATES/AUDIT_EVIDENCE_TEMPLATE.md` como plantilla estándar para registrar evidencia de auditorías, certificaciones, revisiones de seguridad, QA, arquitectura, producción, agentes, skills y cumplimiento.

### Alcance esperado

La plantilla deberá incluir, como mínimo:

- Elemento auditado.
- Fuente revisada.
- Tipo de evidencia.
- Evidencia observada.
- Clasificación de verificabilidad.
- Severidad.
- Estado.
- Responsable.
- Observaciones.
- Evidencia externa requerida, si corresponde.
- Criterio GO / GO CON CONDICIONES / NO-GO, si aplica.

### Motivo

Evitar formatos inconsistentes entre auditorías y asegurar trazabilidad documental bajo el principio:

> La ausencia de evidencia no equivale a cumplimiento.

## ADR Futuro — Evidence-First Auditing

**Estado:** Pendiente
**Prioridad:** Media
**Origen:** CTO / Dominio 09
**Relacionado con:** GOV-SEC-001

### Objetivo

Formalizar mediante ADR el principio de auditoría basada en evidencia verificable como decisión permanente de gobierno transversal para Aiagro.

### Ámbitos incluidos

- Seguridad.
- QA.
- Arquitectura.
- Producción.
- Compliance.
- Agentes IA.
- Skills.
- Revisiones CTO.
- Gates de release.

### Motivo

La decisión tiene impacto transversal y debe quedar registrada como estándar permanente de gobierno documental, auditoría y aceptación de avances.
